01 — 全体像

def は、中身の処理をぜんぶ包み込む「箱」そのもの。

関数 ＝ 並んだ３ステップではなく、「箱の中に、空のお皿・加熱・出力がぜんぶ入っている」 イメージ。
箱（ルール）を作ってから、その中で順番に処理が走ります。

DEF · 電子レンジの箱（ルール）

06 — コードで見る

def から return までを、コードで見比べる。

「箱の中に、空のお皿・加熱・出力」がぜんぶ入っているのが、コードの形にも現れます。

warm_up.py
# ▼ 【レンジの箱（ルール）】 名前と入口を決める
def 温め(menu_list):

    # ▼ ① 空のお皿をセット
    hot_menu = 0

    # ▼ ② 加熱中（一つずつ温めて積み上げる）
    for 具材 in menu_list:
        hot_menu = hot_menu + 具材

    # ▼ ③ チーン！完成したお皿を外へ
    return hot_menu


# ── 呼び出し側 ──────────────
lunch_set = [10, 20, 30]
結果 = 温め(lunch_set)
DEF · 箱（ルール）

def 温め(menu_list): で「温め」という箱と入口を作る。
この中身がぜんぶ "レンジの内側"。

① SET · 空のお皿

hot_menu = 0 で、温めたものを乗せていく
受け皿を中に用意する。

② HEATING · 一つずつ温める

for 具材 in menu_list: で、
具材を順番に加熱し、お皿に積み上げる。

③ RETURN · チーン！

return hot_menu で、完成したお皿を
呼び出し元（結果）へ放り出す。

07 — 引数の名前の違い（バトンタッチ）

同じ中身に、外と中で違う名札。
箱に入った瞬間、名前が バトンタッチ される。

外の世界では lunch_set（具体的な実物データ）と呼んでいるものを、
レンジ（関数）の中に入った瞬間、設計図通りの呼び名 menu_list として受け取り直します。

08 — まとめ

覚えて帰る、関数の４つの顔。

def という「箱」の中に、空のお皿・加熱・出力がぜんぶ入っている。
引数の名前は、外と中で違っていて当たり前（バトンタッチ）。

STEP 01
def 温め(menu_list):レンジの箱
関数の名前と入口を決める。
中身を包む「箱」を宣言するだけ。
STEP 02
hot_menu = 0空のお皿
温めたものを乗せていく
"受け皿" を箱の中に用意。
STEP 03
for 具材 in menu_list:一つずつ温める
具材を 順番に加熱 し、
お皿に積み上げる繰り返し作業。
STEP 04
return hot_menuチーン！
完成したお皿を 呼び出し元へ 放り出す。これが無いと外に出ない。
TAKEAWAY
外の lunch_set は実物、中の menu_list は仮の呼び名。
箱の入口でバトンタッチして、レンジは中身を選ばずに動く。
質問や不明点があればチャットで聞いてください。
